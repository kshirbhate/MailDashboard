UI Review Checklist

  • Visuals
    • UI elements should  render exactly same as in design mockups

      • Check for extra paddings

        • This might change div's height and width

        Check for extra margins

        • This might give you white space in between divs.

        Check for horinzontal scrollbar.

        • This comes because there might be a width, padding or margin that is going outside to parent div.

        Check for Typography elements

        • Check for applied styles to elements, this should be as per markups
          • Colors
          • Font-weights
          • size
          • height
          • width

        Check for overlay (popovers, modals, tooltips)

        • when checking on mobile, width should not go outside viewport for overlays.

        If using parent child elements, then calculate padding, margins, width or height properly so that parent div will have proper style in terms of padding, margins, width or height.

        Check whether same classes are not being rendered in style tab of inspect element.

        • This happens because of importing same css file in components which are again rendered in parent component.

        Try to avoid inline styles, instead create feature level styles and override

        • If you want to override a style then you should write class specific to that element and put that in feature level css file.

        Give id's to each elements and they should be unique

        • You can apply styles by using ids. but you should give styles to ids only wherever required. because id has more weightage than class..

        When using third party css library look into this points

        • whether it overrides your style classes
        • When using row, col classes check for paddings and margins.
    • Check for spelling and grammar errors
    • remove console.logs statements from code.
    • This above steps can be done on Chrome, Mozilla, IE for Desktop, Mobile and Tablet also
  • Code
    • UI Styles
      • When writing style classes, set your browsers on minimum boundaries
        • Set 320 for mobile
        • Set 768 for tablet
        • Set 992 for desktop
      • Consistent naming conventions are used
      • CSS validates against the W3C validator
      • CSS selectors are only as specific as they need to be; grouped logically
      • Events and styles applied to :hover are also applied to :focus
    • UI Code
      • Code does not contain inline style attributes
      • Code does not contain deprecated elements & attributes
      • Tags and attributes are lowercase (For CSS)
      • Tags are closed and nested properly
      • Tables are only used to display tabular data
      • Element IDs are unique
      • Add Comments wherever required.
      • Their should not be any garbage code.
      • Import only those third party libraries which are required.
      • Code should run with 0 errors in console.
      • Code should run with 0 warnings in console
